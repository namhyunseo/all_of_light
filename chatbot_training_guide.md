# Titan 12.0 챗봇 학습 가이드

Titan 12.0 매뉴얼을 기반으로 작성된 이 가이드는 Titan 생태계와 관련하여 사용자에게 도움을 줄 수 있는 챗봇 학습용 구조화된 정보를 제공합니다.

## 1. 제품 개요 (Product Overview)

- **정의**: Titan 12.0은 Avolites 조명 콘솔 전반에서 사용되는 조명 제어 소프트웨어 운영 체제입니다.
- **주요 목적**: 라이브 엔터테인먼트 환경에서 조명기(dimmer), 무빙 라이트(moving light), 미디어 서버를 제어하기 위해 설계되었습니다. 조명 큐(cue), 체이스(chase), 이펙트(effect)의 생성, 저장 및 재생을 관리합니다.
- **타겟 사용자**: 투어 공연, 페스티벌, 극장, 클럽 등의 조명 디자이너, 프로그래머 및 오퍼레이터.
- **지원 하드웨어**: Sapphire Touch, Arena, Tiger Touch II, Quartz, Titan Mobile, T1/T2 USB 인터페이스, Titan Net Processor (TNP) 등 특정 하드웨어 콘솔에서 구동됩니다.

## 2. 주요 기능 (Key Features)

챗봇은 다음의 핵심 기능들을 설명할 수 있어야 합니다:

- **패치(Patching)**: 조명기(dimmer)와 지능형 조명 기구(intelligent fixture)를 DMX 주소 및 제어 핸들(handle)에 매핑하는 기능 (RDM 패치 및 연결 끊긴 조명 기구 찾기 포함).
- **시각화(Visualisation)**: 조명 리그(rig)의 실시간 시뮬레이션 및 사전 프로그래밍을 위한 통합 Capture Visualiser 제공.
- **조명 기구 제어(Fixture Control)**: 휠을 사용하여 강도(Intensity), 위치(Position), 색상(Colour), 고보(Gobo), 빔(Beam) 등의 속성(IPCGBES) 제어.
- **팔레트(Palettes)**: 자주 사용하는 설정(예: 특정 색상 파렛트, 위치 파렛트)을 저장하여 빠른 호출 및 효율적인 프로그래밍 지원.
- **이펙트 생성(Effects Generation)**:
  - _쉐이프 생성기(Shape Generator):_ 자동 움직임이나 속성 변조 생성 (예: 원형 이동, 나선형 효과).
  - _픽셀 매퍼(Pixel Mapper):_ 비디오나 이미지를 조명 기구 그리드에 매핑하여 효과 연출.
  - _키 프레임 쉐이프(Key Frame Shapes):_ 사용자 정의 이펙트 시퀀스 생성 가능.
- **프로그래밍 객체(Programming Objects)**:
  - _큐(Cues):_ 단일 정적 조명 장면.
  - _체이스(Chases):_ 시간 간격을 둔 큐의 시퀀스 (순차 재생).
  - _큐 리스트(Cue Lists):_ 수동 또는 타임코드로 트리거되는 극장식 큐 목록.
- **네트워킹(TitanNet)**: DMX 라인을 최대 64 유니버스까지 확장 가능하며, 다중 사용자 작업 및 백업 이중화 지원.
- **시너지(Synergy)**: Ai 미디어 서버와 통합하여 비디오 레이어를 제어하고 비디오 콘텐츠를 조명 기구에 매핑.

## 3. 기술 사양 (Technical Specifications)

- **소프트웨어 플랫폼**: Titan은 임베디드 Windows 운영 체제(Windows 7, 8.1 또는 10, PC Suite의 경우 64비트 필요)에서 실행됩니다.
- **DMX 용량**:
  - 콘솔 자체적으로 최대 16 DMX 유니버스 처리 가능.
  - T1 인터페이스는 1 유니버스, T2는 2 유니버스로 제한.
  - Titan Net Processor (TNP)를 사용하여 최대 64 유니버스까지 시스템 확장 가능.
- **출력(Outputs)**:
  - 물리적 5핀 XLR DMX 출력 (콘솔 모델에 따라 수량 상이).
  - 이더넷 프로토콜: Art-Net 및 sACN 지원.
- **하드웨어 인터페이스**:
  - 터치스크린 (내장 및 외장 모니터 지원).
  - 물리적 재생 페이더 및 인코더 (휠).
  - 광학 네트워크 연결 (Arena 콘솔 전용).
  - MIDI 및 SMPTE 타임코드 입력 지원.

## 4. 설치 및 설정 (Installation & Setup)

### 전원 켜기 (Powering Up)

1. **연결**: 주전원(80-260V)을 연결합니다. 내부 UPS가 없는 콘솔의 경우 외부 UPS 사용을 권장합니다.
2. **부팅**: 콘솔 전면 또는 패널의 전원 스위치를 누릅니다. 부팅 시간은 약 1분 30초 소요됩니다.
3. **종료**: 전원 스위치를 짧게 누르거나 소프트웨어 메뉴를 통해 종료합니다. 데이터 손실 방지를 위해 전원을 바로 차단하지 마십시오.

### PC Suite 연결 (Titan Mobile/T1/T2)

1. **드라이버 설치**: 하드웨어를 연결하기 _전에_ Titan PC Suite 소프트웨어를 설치하여 드라이버가 제대로 인식되도록 합니다.
2. **라이선스**: AvoKey USB 동글이 필요합니다. Authenticator 마법사를 통해 인증을 진행합니다.

### 네트워크 설정 (Network Setup)

1. **IP 주소**: System 모드 > [Network Settings]에서 설정합니다. 통신을 위해 콘솔들은 동일한 IP 대역(예: Art-Net의 경우 10.x.x.x) 및 서브넷 마스크를 사용해야 합니다.
2. **DMX 출력**: System 모드 > [DMX Settings]에서 DMX 라인을 물리적 노드 또는 Art-Net/sACN으로 구성합니다.

## 5. 운영 지침 (Operating Instructions)

### 조명 기구 패치 (Patching Fixtures)

1. `<Patch>` 버튼을 누른 후 [Fixtures]를 선택합니다.
2. 소프트키에서 제조사와 모델을 선택합니다.
3. DMX 라인과 주소를 설정합니다.
4. 파란색 선택 버튼(핸들)을 눌러 해당 페이더/버튼에 조명 기구를 패치합니다.

### 조명 기구 제어 (Controlling Fixtures)

1. **선택**: 파란색 조명 기구 버튼을 눌러 선택합니다.
2. **로케이트(Locate)**: `<Locate>` 버튼을 눌러 조명 기구를 켜고 위치를 중앙으로 초기화합니다.
3. **조정**: 속성 뱅크 버튼(Intensity, Colour, Position 등)과 로터리 휠을 사용하여 파라미터를 조정합니다.
4. **클리어(Clear)**: `<Clear>` 버튼을 눌러 프로그래머(Programmer)에서 활성 값을 제거합니다.

### 큐 기록 (Recording a Cue)

1. 무대 조명 상태를 설정합니다.
2. `<Record>` 버튼을 누릅니다.
3. 비어 있는 재생 페이더(Swop 버튼) 또는 터치 버튼을 선택합니다.
4. `<Clear>` 버튼을 눌러 작업을 마칩니다.

### 팔레트 사용 (Using Palettes)

1. 조명 기구를 선택하고 특정 속성을 설정합니다 (예: 색상 빨강).
2. `<Record>` 버튼을 누른 다음 `<Palette>`를 누릅니다.
3. 속성 마스크를 선택합니다 (예: Colour만 선택).
4. 작업 공간의 팔레트 버튼을 눌러 저장합니다.

### 빠른 시작 단축키 (Quick Start Shortcuts)

- **Find Fixture**: 설치된 조명의 DMX 주소를 식별하기 위한 "Find Fixture" 모드 제공.
- **Fan Mode**: 선택된 여러 조명 기구에 값(팬/틸트 등)을 부채꼴처럼 펼쳐서 적용.
- **Highlight**: 선택된 조명 기구를 밝게 하고 다른 기구들을 어둡게 하여 위치 조정을 용이하게 함.

## 6. 문제 해결 (Troubleshooting)

- **헬스체크(Healthcheck)**: 시스템 부팅 시 파일 시스템이나 펌웨어 문제를 감지하기 위해 "Titan Healthcheck"를 실행합니다.
- **조명 기구 문제**:
  - _방향 오류:_ "Flip" 기능을 사용하여 팬/틸트 요크 방향을 전환합니다.
  - _주소 오류:_ "Find Fixture"를 사용하여 리그상의 실제 주소를 식별합니다.
  - _이상한 움직임:_ "Fan" 모드가 실수로 켜져 있는지 확인합니다.
- **콘솔 멈춤(Lockup)**:
  - _패널 리셋:_ DMX 처리를 중단하지 않고 전면 패널 전자 장치만 재시작하려면 전용 리셋 스위치(클립 구멍)를 사용합니다.
  - _강제 종료:_ 전원 버튼을 5초간 누릅니다 (마지막 저장 이후 데이터는 손실됨).
- **쇼 호환성**: 최신 버전의 쇼 파일은 구형 소프트웨어에서 로드되지 않을 수 있습니다. 다른 버전을 실행하려면 "Switch Software"를 사용하십시오.

## 7. 안전 및 경고 (Safety & Warnings)

- **종료**: 콘솔 후면의 전원 차단 스위치로 끄지 마십시오. 항상 전면 전원 스위치를 사용하여 소프트웨어가 정상적으로 종료되고 데이터가 저장되도록 해야 합니다.
- **UPS**: 내부 UPS가 있는 콘솔의 경우, 강제 종료를 위해 전원 케이블만 뽑지 마십시오. 하드 리셋이 절대적으로 필요한 경우 배터리 차단 스위치를 사용하십시오.
- **전파 간섭**: 콘솔 위나 근처에 무선 송신기를 두지 마십시오.
- **서비스**: 내부에는 사용자가 수리할 수 있는 부품이 없습니다. 고전압 위험이 있으므로 자격을 갖춘 직원에게 서비스를 의뢰하십시오.
- **데이터 백업**: 내부 UPS가 없는 콘솔의 경우 외부 UPS 사용을 권장합니다. USB 드라이브에 쇼 파일을 자주 저장하십시오.

## 8. 용어 사전 (Glossary of Terms)

- **핸들(Handle)**: 조명 기구 또는 재생을 제어하는 데 사용되는 페이더와 버튼(Swop/Flash/Select)의 조합.
- **프로그래머(Programmer)**: 조명 기구에 대한 수동 변경 사항이 기록되기 전에 임시로 유지되는 메모리 영역.
- **팔레트(Palette)**: 여러 큐에서 재사용할 수 있도록 저장된 특정 속성 값 (예: "빨강" 또는 "중앙 무대").
- **조명 기구 퍼스널리티(Fixture Personality)**: 특정 조명 기구 모델의 DMX 채널 매핑 및 동작을 정의하는 파일.
- **속성(Attribute)**: 조명 기구의 제어 가능한 특정 파라미터 (예: 팬, 틸트, 디머, 고보).
- **IPCGBES**: 속성 그룹의 약어 (Intensity, Position, Colour, Gobo, Beam, Effect, Special).
- **로케이트(Locate)**: 조명 기구를 강도(Intensity) 최대 상태의 "홈" 위치로 설정.
- **LTP (Latest Takes Precedence)**: 가장 최근의 명령이 출력 레벨을 결정하는 제어 로직 (주로 무빙 라이트 속성에 사용).
- **HTP (Highest Takes Precedence)**: 가장 높은 레벨이 출력을 결정하는 제어 로직 (주로 디머에 사용).
- **TNP (Titan Net Processor)**: DMX 처리 용량을 확장하기 위해 사용되는 외부 하드웨어.
- **시너지(Synergy)**: Titan을 Ai 미디어 서버와 연결하는 시스템.
